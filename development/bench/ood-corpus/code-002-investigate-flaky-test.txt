log in. CI output for failing runs shows the polling step times out at 30s.
Local runs of the same test complete in ~2s.

Recent context:
- The CI workers run on shared k8s nodes with a noisy-neighbor problem
- The email-mock service runs as a sidecar container with no resource limits
- Tests were stable until 3 weeks ago when the team upgraded the polling
  library from v2.1 → v2.4

Produce a RANKED list (most likely → least likely) of hypotheses for the
flakiness, with each entry containing:

- `:why-this-cause` — the reasoning chain from observed symptoms
- `:falsifiability-test` — a SPECIFIC experiment that would PROVE this is
