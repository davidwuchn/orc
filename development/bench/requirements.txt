# ColBERT Python venv — pinned, verified-working dependencies.
#
# Captured from the .venv-colbert that LIVE-passed the QP-3 verify on
# 2026-06-08 (Python 3.12), then trimmed to drop transitive deps that
# ragatouille pulls in but the bridge doesn't use at runtime
# (langgraph, llama-index, openai, etc.). The trimmed set is what
# pip can resolve from scratch in one pass; the full freeze hit a
# langgraph-vs-langchain-core conflict that only worked because the
# original install was staged.
#
# Why the load-bearing pins are needed:
#   - ragatouille 0.0.9 imports langchain.retrievers (the pre-0.3 API).
#     Breaks on langchain >= 0.3.
#   - colbert-ai 0.2.22 uses HF_ColBERT.all_tied_weights_keys which
#     transformers 5.x removed. Needs transformers 4.x.
#   - fast-pytorch-kmeans uses psutil without declaring it.
#   - langchain 0.2.x needs numpy < 2.0.
#
# A bare `pip install ragatouille>=0.0.9 torch>=2.0` under today's pip
# resolver picks newest-compatible-on-paper versions and hits all three
# bugs above. This pin file gives pip the constraints it needs to land
# on a working set.
#
# To use:
#   bash scripts/setup-colbert.sh
#
# To regenerate after upgrading a dep, freeze ONLY the load-bearing set
# (don't freeze the full env or langgraph/llama-index will re-enter):
#   .venv-colbert/bin/pip freeze | grep -E '^(ragatouille|colbert-ai|transformers|tokenizers|huggingface_hub|langchain|langchain-community|langchain-core|psutil|numpy|tenacity|sentence-transformers|torch|faiss-cpu|fast_pytorch_kmeans|ninja|onnx)==' > /tmp/freeze.txt
